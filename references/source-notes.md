# Source Notes

This plugin is derived from the user's local EPUB copy of *The Book of Elon* by Eric Jorgenson.

Do not copy long passages from the book into answers, skills, docs, or examples. Use paraphrased method summaries and cite section names when useful.

## Attribution Style

Use language like:

- "Book-derived method"
- "Inspired by the book section..."
- "The plugin maps this to..."

Avoid language like:

- "Elon says to always..."
- "This proves..."
- "This is the only correct method..."

## What The Plugin Preserves

- Reusable workflows.
- Decision lenses.
- Method names and section-level maps.
- Examples of when to apply each method.

## What The Plugin Avoids

- Long quote collections.
- Full chapter summaries inside every skill.
- Claims that the methods are universally right.
- Turning every sentence into a separate top-level skill.

## Safe Source Use

When the user asks where a method came from, refer to `book-map.md`. When the user asks for the full set of ideas, refer to `method-catalog.md`. When the user asks for reading recommendations, use `x-reading`.
