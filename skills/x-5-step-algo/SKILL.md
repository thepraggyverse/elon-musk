# X 5 Step Algo

Use this skill before optimizing or automating. The order matters.

## Covers

- Question every requirement.
- Treat requirements as recommendations until an accountable owner proves their necessity.
- Delete the part or process.
- The best part is no part when the part is unnecessary.
- Simplify and optimize what remains.
- Use simplicity to improve reliability and low cost.
- Find the design necessity of every surviving part and process.
- Accelerate cycle time.
- Automate last.
- Overdelete and add back only what evidence proves necessary.

## Workflow

1. Question requirements: treat them as recommendations and name the owner, reason, evidence, and current validity.
2. Delete: remove features, steps, fields, meetings, approvals, code paths, or tools that do not prove necessity.
3. Simplify: state each survivor's design necessity and make the remaining path reliable, low-cost, direct, reversible, and inspectable.
4. Accelerate: shorten the loop only after the direction is valid.
5. Automate: automate stable repeated work with visible failure modes.

## Output

```text
5 Step Algo:
1. Requirements:
2. Delete:
3. Simplify:
4. Accelerate:
5. Automate:
Add-back trigger:
Recommendation:
Next action:
```

## Completion Gate

Complete only when all five steps appear in order, every surviving requirement has a necessity basis, and deletion has an evidence-based add-back trigger.

## Example

Use `x-5-step-algo` before building a dashboard: "Which requirements are real, what can be deleted, and what should not be automated yet?"
