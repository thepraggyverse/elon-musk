# X Urgency

Use this skill to make progress faster after the direction is worth pursuing. If the requirements are suspect, run `x-5-step-algo` first.

## Covers

- Do not waste time.
- Speed is offense and defense.
- Do things in parallel when dependencies allow.
- Break down the impossible.
- Set aggressive timelines while admitting uncertainty.
- Use a visible scoreboard for progress.

## Workflow

1. Name the deadline and why speed matters.
2. Find serial work that can become parallel.
3. Identify the real dependency chain.
4. Break the "impossible" into parts, resources, constraints, and first proof.
5. Set a shorter learning loop with a clear score.
6. Define rollback or stop conditions.

## Output

```text
Urgency Review:
Why speed matters:
Critical path:
Parallel work:
Impossible breakdown:
Scoreboard:
Timeline:
Stop / rollback:
```

## Example

Use `x-urgency` on a six-month migration: "What can we test this week that reveals whether the plan works?"
