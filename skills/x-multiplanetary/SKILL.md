# X Multiplanetary

Use this skill when the mission is huge enough that normal roadmap thinking feels too small.

## Covers

- Make life or a mission resilient beyond one fragile base.
- Treat the first gateway as the key enabling milestone.
- Break the impossible into staged capability thresholds.
- Optimize every subsystem toward the mission metric.
- Build the new world after reaching the gateway.

## Workflow

1. Name the mission and the reason it matters.
2. Define the gateway milestone that unlocks the rest.
3. Work backward into capability thresholds.
4. Identify the mission metric equivalent of "mass to Mars."
5. Separate survival infrastructure from optional expansion.
6. Produce a staged roadmap.

## Output

```text
Multiplanetary Roadmap:
Mission:
Gateway milestone:
Capability thresholds:
Mission metric:
Survival infrastructure:
Stages:
Next proof:
```

## Completion Gate

Complete only when the mission metric, gateway proof, every capability threshold, survival infrastructure, resilience value, and immediate next proof are explicit.

## Example

Use `x-multiplanetary` on a personal knowledge system: "How do we make this survive one machine, one app, or one vendor failing?"
