# X Organization

Use this skill when the system is slow because people cannot talk, decide, learn, or fail cleanly.

## Covers

- Remove organizational boundaries.
- Let people talk directly when it helps the company.
- Use simple language.
- Permit non-catastrophic failure and fast learning.
- Delete meetings, approvals, and handoffs that do not improve the product or service.

## Workflow

1. Map the decision or workflow path.
2. Mark every handoff, approval, meeting, status artifact, and translation layer.
3. Ask which steps protect a real risk and which steps protect comfort or hierarchy.
4. Replace indirect communication with direct owner-to-owner loops.
5. Simplify language until the work is clear to a competent outsider.
6. Define where failure is acceptable and where it is not.

## Output

```text

```

## Example

Use `x-org` on a blocked launch: "Which approvals are real controls, and which are just accumulated process?"
